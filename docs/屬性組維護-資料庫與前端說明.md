# 屬性組維護：資料庫與前端說明

本文件說明「屬性組維護＆全域字典主檔」功能所使用之資料庫四張表、後端 API、以及前端頁面與欄位對齊方式。前後端欄位與資料庫對齊，不做額外映射。

---

## 一、資料庫（四張表）

資料庫：`MIDDB_EC_DEV`（或專案 `config` 所設之 database），Schema：`dbo`。

### 1.1 INVMI（屬性主檔）

| 欄位   | 說明           | 範例 |
|--------|----------------|------|
| MI001  | 屬性代碼（鍵） | 01, 04, 05, 06 |
| MI002  | 中文定義       | 識別碼、產品檔代碼、客戶、流水號保留碼 |
| MI003  | 樣式／長度     | 1, 4, 2, 1 |
| MI004  | 引用／序號     | 1 |

用途：定義「有哪些屬性」（每一筆為一種屬性代碼及其說明）。

### 1.2 INVMJ（屬性值明細）

| 欄位   | 說明                 | 範例 |
|--------|----------------------|------|
| MJ001  | 屬性代碼（對應 MI001） | 01, 04 |
| MJ002  | 序號                 | 0001, 0002, … |
| MJ003  | 值代碼               | 2, 3, 5 或 BAST, CGST, … |
| MJ004  | 值說明（label）      | 半成品、管料、零件、BATTERY SPRAY PAINTING 電池蓋烤漆… |

用途：某一屬性（MJ001 = MI001）下的可選值；一筆為一個選項（value = MJ003, label = MJ004）。

### 1.3 INVMK（屬性組／規則主檔）

| 欄位   | 說明           | 範例 |
|--------|----------------|------|
| MK001  | 組別 ID        | 204 |
| MK002  | 組名／樣式描述 | 半成品+4碼產品代碼 |
| MK003～MK009 | 旗標（Y/N） | Y, N, … |
| MK010  | 數值           | 6 |

用途：一筆代表一個「屬性組」或「規則」定義。目前後端「屬性組規則」列表為以 INVMI 為主之實作，未使用 INVMK。

### 1.4 INVML（屬性組明細）

| 欄位   | 說明                 | 範例 |
|--------|----------------------|------|
| ML001  | 組別 ID（對應 MK001） | 204 |
| ML002  | 序號                 | 0001, 0002, 0003, 0004 |
| ML003  | 屬性代碼（對應 MI001） | 01, 04, 05, 06 |
| ML004～ML006 | 旗標（Y/N）   | Y, N |

用途：描述某一組（ML001）由哪些屬性（ML003）組成。目前後端未使用 INVML。

### 1.5 表關聯摘要

```
INVMI (屬性主檔)
  MI001 ←── INVMJ.MJ001（某屬性下的選項）
  MI001 ←── INVML.ML003（某組包含的屬性）

INVMK (屬性組主檔)
  MK001 ←── INVML.ML001（某組的明細）
```

目前後端實際使用：**INVMI**（規則列表、代碼清單、單筆／詳情）、**INVMJ**（依 MI001 取屬性值列表）。INVMK、INVML 未使用。

## 二、後端 API

專案路徑：`backend/`。屬性組維護模組：`src/routes/attributeGroup.routes.js`、`src/controllers/attributeGroup.controller.js`、`src/services/attributeGroup.service.js`。

### 2.1 路由與對應表

| 方法 | 路徑 | 說明 | 使用表 |
|------|------|------|--------|
| GET | /api/attributeGroup/rules | 屬性組規則列表，可選 `?keyword=` | INVMI |
| GET | /api/attributeGroup/rules/:code | 依 code 取得單筆規則 | INVMI |
| GET | /api/attributeGroup/dictionary | 屬性代碼清單，可選 `?lang=` | INVMI |
| GET | /api/attributeGroup/dictionary/:code | 單一代碼詳情＋屬性值列表 | INVMI + INVMJ |

### 2.2 回應格式

- 成功：`{ success: true, data: ... , message: "..." }`
- 失敗：`{ success: false, error: "..." , message: "..." }`，HTTP 4xx/5xx

### 2.3 各 API 回傳欄位（與前端對齊）

**GET /api/attributeGroup/rules**（陣列）

| 欄位 | 來源 | 說明 |
|------|------|------|
| code | INVMI.MI001 | 屬性代碼 |
| definition | INVMI.MI002 | 定義 |
| style | INVMI.MI003 | 樣式 |
| serialNumber | INVMI.MI004 | 流水碼 |
| sapModel | 固定 'ALL' | SAP MODEL（無對應表時固定） |
| status | 固定 '啟用' | 狀態（無對應表時固定） |

**GET /api/attributeGroup/rules/:code**（單筆）：同上欄位。

**GET /api/attributeGroup/dictionary**（陣列）

| 欄位 | 來源 | 說明 |
|------|------|------|
| code | INVMI.MI001 | 代碼 |
| definition | INVMI.MI002 | 定義 |
| refs | INVMI.MI004 | 引用 |

**GET /api/attributeGroup/dictionary/:code**（單筆）

| 欄位 | 來源 | 說明 |
|------|------|------|
| code | INVMI.MI001 | 代碼 |
| definition | INVMI.MI002 | 定義 |
| refs | INVMI.MI004 | 引用 |
| values | INVMJ（MJ001=MI001） | 陣列，每項 { value: MJ003, label: MJ004 } |

## 三、前端實作

專案路徑：`fronted/`。頁面：屬性組維護（路由 `/attribute-group-maintenance`）。

### 3.1 路由與選單

- 路由：`fronted/src/router/index.js`，path：`/attribute-group-maintenance`，name：`AttributeGroupMaintenance`，component：`AttributeGroupMaintenance.vue`。
- 側欄：`fronted/src/components/SidebarMenu.vue`，於「料號服務」下「屬性群組維護」連結至上述 path。

### 3.2 API 模組

檔案：`fronted/src/api/attributeGroup.js`。

- `getRulesList(keyword)` → GET `/attributeGroup/rules`，可帶 `keyword`。
- `getRuleByCode(code)` → GET `/attributeGroup/rules/:code`。
- `getDictionaryList(lang)` → GET `/attributeGroup/dictionary`，可帶 `lang`（zh-TW / en）。
- `getDictionaryByCode(code)` → GET `/attributeGroup/dictionary/:code`。

使用 `fronted/src/api/request.js` 的 `get()`，基底 URL 為 `VITE_API_BASE_URL`（預設 `http://localhost:3000/api`）。回傳為後端整包 `{ success, data, message }`。

### 3.3 頁面結構（AttributeGroupMaintenance.vue）

- **頂部**：標題「屬性組維護＆全域字典主檔」、副標題；右側「語言」「SAP 模板」下拉（語言變更會重載字典清單）。
- **Tab 1：屬性組規則**
  - 搜尋輸入框、搜尋／新增屬性組／重置按鈕。
  - 表格欄位：code, definition, style, serialNumber, sapModel, status，操作（編輯）。
  - 資料來源：`getRulesList(rulesSearchKeyword)`，結果賦予 `rulesList`，直接綁定表格，不另做欄位映射。
- **Tab 2：全域字典主檔**
  - 左：屬性代碼清單（表格 code, definition, refs，操作 X）；資料來源：`getDictionaryList(selectedLanguage)` → `dictionaryCodeList`。
  - 右：屬性值維護；選取左表一列時呼叫 `getDictionaryByCode(row.code)`，將 `res.data.values` 賦予 `selectedCodeValues`，表格欄位為 value、label，不另做映射。

### 3.4 欄位對齊原則

前端**直接使用後端回傳之欄位名稱**，不重新命名、不轉換：

- 規則表：`row.code`, `row.definition`, `row.style`, `row.serialNumber`, `row.sapModel`, `row.status`。
- 代碼清單：`row.code`, `row.definition`, `row.refs`。
- 屬性值：`row.value`, `row.label`（來自後端 `values[]`）。

因此資料流為：**資料庫欄位 → 後端 SELECT/AS → 前端綁定**，僅在後端做一次 AS（如 MI001 AS code），前端不再映射。
